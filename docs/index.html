Hospital Inventory Management System

This is a comprehensive inventory management system designed for hospital to efficiently track and manage their medical supplies, including medicines and other essential items. The system is built using the TALL (TailwindCSS, inertiajs/react, Laravel) stack, providing a modern and reactive user interface.

Features

  • Medicine Management:
    • Add, edit, and delete medicine records.
    • Track medicine stock levels.
    • Record medicine expiration dates.
    • View medicine usage logs.
  • Generic Management:
    • Add, edit, and delete generic drug information.
    • Associate generics with specific medicines.
  • Ward Management:
    • Define and manage different wards or locations within the hospital/clinics.
    • can add ward size.
  • Expense Management:
    • Record various expenses related to hospital operations.
    • Categorize expenses.
    • Track expense records.
  • User Authentication and Authorization:
    • Secure user login and registration.
    • Role-based access control to different parts of the system. (Future Feature)
  • Backup Functionality:
    • Scheduled and manual database backups. (some errors but will fix soon)
  • Reporting: (Future Feature)
    • Generate reports on medicine usage, stock levels, and expenses.

Technologies Used

  • Backend: Laravel (PHP Framework)
  • Frontend:
    • TailwindCSS (CSS Framework)
    • Inertiajs (JavaScript Framework)
    • Shadcn UI (Component laibray for react)
  • Database: MySQL

Installation

Prerequisites

  • PHP >= 8.3
  • Composer
  • Node.js & npm
  • MySQL Database

Setup Steps

  1. Clone the repository: 
    git clone <repository_url>
cd <project_directory>
  2. Install Composer dependencies: 
    composer install
  3. Install Node.js dependencies: 
    npm install
  4. Copy the environment file: 
    cp .env.example .env
  5. Configure your database in the .env file: 
    DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_database_user
DB_PASSWORD=your_database_password
  6. Generate an application key: 
    php artisan key:generate
  7. Run database migrations: 
    php artisan migrate
  8. (Optional) Seed the database: 
    php artisan db:seed
  9. Compile frontend assets: 
    npm run dev

    For production, use:

    npm run build
  10. Start the local development server: 
    php artisan serve

You should now be able to access the application at http://127.0.0.1:8000.

Usage

Once installed and running, navigate to the application in your web browser. You will be presented with the login or registration page. After logging in, you can access the different modules (Medicines, Generics, Wards, Expenses) through the navigation menu to manage your clinic's inventory.
